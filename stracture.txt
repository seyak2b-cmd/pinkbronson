■ 概要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Twitch配信者向けのプロデューサーデスクシステム。
メインハブ (pink_bronson.py) が STT と子プロセス管理を担い、
3つのサブツールを子プロセスとして起動する。

  pink_bronson.py              … メインハブ (STT + 子プロセス起動)
  └── Blue_Rayban/             … 翻訳エンジン + Firebase中継
  └── Emerald_Rolex/           … Twitchチャット取得 + Firebase
  └── Golden_Chain/            … 会話要約 + タイトル/ファシリテーター生成

外部サービス: Twitch IRC, Twitch API (Helix), Gemini AI, Firebase Realtime DB,
              VOICEVOX (ローカル), seya-chat-trans.web.app (Firebaseホスティング)

■ データフロー
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[マイク音声]
  → pink_bronson.py (faster-whisper STT)
  → data/stt_bridge.json
  → STTWatcher (Blue_Rayban/mainTST.py)
  → [英訳] → Firebase /stt_en, /stt_history
  → [Web TTS] → Firebase /config/tts_audio → seya-chat-trans.web.app で再生

[Twitch IRC]
  → Emerald_Rolex (emerald_rolex.py) ← チャット処理の唯一の担当
  → [翻訳(JA/EN)] → Firebase /chats.json
  → Blue_Rayban/mainTST.py (twitchio Bot) ← 受信ログのみ (翻訳・push なし)

[mobile.html ビューア投稿]
  → Firebase /viewer_queue/{key}
  → ViewerQueueWatcher (Blue_Rayban/mainTST.py)
  → Firebase /chats + Twitchチャットへ転送

[Golden_Chain 出力]
  → output/summary.txt, title.txt, facilitator.txt
  → GoldenChainWatcher (Blue_Rayban/mainTST.py)
  → Firebase /golden_chain/{summary|title|facilitator}
  → seya-chat-trans.web.app で表示

[各モジュール]
  → system_logger.send_system_log()
  → Firebase /system_logs
  → monitor.html で確認可能

  /system_logs.json          システムログ (全モジュール)
